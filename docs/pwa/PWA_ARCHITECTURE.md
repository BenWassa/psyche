## Request/Response Lifecycle

```
User Request
    │
    ▼
┌─────────────────────────────────┐
│  Service Worker Intercepts      │
└─────────────────────────────────┘
    │
    ├──────────────────────────────────────────────┐
    │                                              │
    ▼                                              ▼
Is cached?                                    Check Network
(by URL)                                            │
    │                                              ▼
    ├─ YES (Static Asset)                    GET from Server
    │   └─► Return from Cache                      │
    │       + Update if stale                      ▼
    │       └─► Cache new version             Store in Cache
    │           (background)                       │
    │                                              ▼
    ├─ NO (HTML/Dynamic)                     Return to App
    │   ├─ Try network first
    │   └─ Fallback to cache if offline
    │
    └─► Response to Browser
            │
            ▼
    ┌──────────────────┐
    │  Browser Renders │
    └──────────────────┘
```

## File Hashing Strategy

```
Build Process:

src/App.tsx
src/main.tsx
src/App.css
    │
    ├─ Webpack/Rollup hashing
    │
    ▼
dist/app-a1b2c3d4.js   (Hash changes if code changes)
dist/app-xyz789ff.css  (Hash changes if styles change)
dist/vendor-abc123.js  (Stable hash if no vendor changes)

Service Worker sees:
  • Old version: app-a1b2c3d4.js
  • New version: app-x5y6z7w8.js
  • DIFFERENT hash → New version detected!
  • Downloads new assets automatically
  • Users get new code seamlessly
```

## Update Detection & Installation

```
Timeline of PWA Update:

t=0s    User has v1.0 installed & running
        Service Worker: "I'm controlling v1.0"

t=60s   Service Worker background check runs
        "Is there a new version on the server?"
        Fetches manifest.json to detect changes

t=62s   ✅ New version detected!
        v2.0 found on server
        Downloaded in background (non-blocking)

t=63s   New service worker registered
        Enters "waiting" state
        App notifies user: "Update Available"

t=X     User clicks "Update" button
        Sends: { type: 'SKIP_WAITING' }
        New service worker takes over
        Page reloads
        
t=X+2s  ✅ App now running v2.0
        User sees new features/fixes
        No interruption to workflow
```

## Caching Strategy Details

```
┌─────────────────────────────────────────────┐
│         Caching Decision Matrix              │
├──────────────┬──────────┬────────────────────┤
│ Resource     │ Strategy │ Cache Duration     │
├──────────────┼──────────┼────────────────────┤
│ google-fonts │ Cache    │ 1 year             │
│ CSS/JS       │ First    │ Immutable (hash)   │
│ Images       │ Cache    │ 30 days            │
│ Icons        │ First    │ 30 days            │
├──────────────┼──────────┼────────────────────┤
│ manifest.json│ Network  │ 1 hour TTL         │
│ HTML         │ First    │ 1 hour             │
│ API Data     │ Network  │ 1 hour             │
│ (future)     │ First    │                    │
├──────────────┼──────────┼────────────────────┤
│ SW JS        │ Network  │ Always             │
│ (never cache)│ First    │ Revalidate         │
└──────────────┴──────────┴────────────────────┘

CacheFirst   = Use cache, update in background
NetworkFirst = Try network, fallback to cache
Always Fresh = Never cache (service worker)
```

## Common Tasks

### Enable Service Worker in Dev
```bash
SW_DEV=true npm run dev
```

### Clear All Caches (Browser)
```javascript
navigator.serviceWorker.getRegistrations().then(regs => {
  regs.forEach(r => r.unregister());
});
caches.keys().then(names => {
  names.forEach(name => caches.delete(name));
});
```

### Check Service Worker Status
```javascript
navigator.serviceWorker.getRegistrations()
  .then(regs => {
    regs.forEach(r => console.log(r.scope, r.active?.state));
  });
```

### View Cache Contents
```javascript
caches.keys().then(names => {
  names.forEach(name => {
    caches.open(name).then(c => c.keys().then(keys => {
      console.log(`${name}: ${keys.length} items`);
      keys.forEach(k => console.log('  -', k.url));
    }));
  });
});
```

### Monitor Updates
```javascript
navigator.serviceWorker.ready.then(reg => {
  reg.addEventListener('updatefound', () => {
    console.log('Update found!');
  });
  setInterval(() => reg.update(), 60000); // Check every minute
});
```
